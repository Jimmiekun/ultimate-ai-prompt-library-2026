# Style Guide

# The Ultimate AI Prompt Library (2026 Edition)

---

## Purpose

This document defines the writing standards for everyone contributing to this project.

All content must follow these rules to ensure the ebook feels like it was written by one author.

---

# Tone

The writing should always be:

- Professional
- Friendly
- Clear
- Practical
- Encouraging
- Beginner-friendly

Avoid overly technical language unless necessary.

---

# Voice

Write as an experienced AI productivity coach.

Teach readers.

Don't lecture them.

Don't assume they already understand AI.

---

# Writing Rules

Use short paragraphs.

Avoid unnecessary jargon.

Explain new concepts simply.

Prefer examples over theory.

Focus on solving problems.

---

# Prompt Writing Standards

Every prompt must:

- Solve one clear problem.
- Produce a useful result.
- Be easy to customize.
- Work across ChatGPT, Claude, and Gemini whenever possible.
- Include placeholders like [Topic], [Business Name], or [Goal] instead of fixed values.

---

# Formatting Standards

Use Markdown headings consistently.

Example:

# Chapter Title

## Section

### Prompt Title

Never skip heading levels.

---

# Lists

Use bullet points whenever possible.

Keep lists concise.

---

# Code Blocks

All copyable prompts should be inside fenced code blocks.

Example:

```text
Act as an expert marketing strategist.

Create a marketing plan for...

```

---

# Examples

Whenever a concept is introduced, include a simple real-world example.

---

# Repetition

Avoid repeating the same explanation in multiple chapters.

If a topic is already explained elsewhere, reference it instead of rewriting it.

---

# Quality Checklist

Before approving any content, ask:

- Is this useful?
- Is it accurate?
- Is it beginner-friendly?
- Is it practical?
- Is it easy to copy and use?
- Does it feel premium?

If any answer is "No", revise the content.

---

# Final Principle

Every page should provide value.

Readers should feel they learned something new or gained a useful tool before turning the page.
